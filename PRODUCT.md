# Product

## Register

brand

## Users

Remove.Simply is for people who need clean image background removal without uploading source files to a web service: makers preparing product shots, creators cleaning assets, and developers or privacy-conscious users who prefer local desktop tools. On the public site, visitors are deciding whether the app is simple enough to download and trustworthy enough to run on their machine.

## Product Purpose

Remove.Simply is a self-contained Electron desktop app for removing image backgrounds locally. It lets users drop or paste images, run local model inference, preview results, and export PNG, WebP, or JPG files. The public page should explain the app quickly, make the privacy/local-processing benefit concrete, and route visitors to the latest GitHub release downloads.

## Brand Personality

Calm, precise, and private. The brand should feel modern and polished, with a restrained header inspired by Perplexity's API Platform page, but it should stay specific to a local desktop image tool rather than generic SaaS. Copy should be direct and practical.

## Anti-references

Avoid loud SaaS hero templates, fake metrics, generic AI buzzwords, gradient text, repetitive icon-card grids, and copy that overstates the app. Avoid making the landing page look like a developer terminal or a generic cloud API product. The app's blue-violet layered mark is the primary identity anchor.

## Design Principles

1. Lead with local trust: make it obvious that image processing happens on the user's machine.
2. Show the workflow in one glance: import, remove, export should read without a long scroll.
3. Keep download paths current: fetch the latest GitHub release and expose platform-specific assets clearly.
4. Preserve quiet polish: use the existing blue-violet brand, precise spacing, and restrained motion.
5. Prioritize clarity over hype: every claim should map to a concrete app capability.

## Accessibility & Inclusion

Target WCAG AA contrast, visible focus states, keyboard-accessible download interactions, semantic landmarks, and reduced-motion alternatives. Motion can be subtle, but the page must work fully with reduced motion enabled.
